Compartir carpetas

Selecciona un tema:

La compartición de archivos en una red es fundamental para la colaboración entre equipos y la gestión eficiente de los recursos. En un entorno de red, compartir carpetas y archivos facilita el intercambio de datos, la centralización de información y el acceso remoto.

En un sistema operativo como Linux, existen diversas formas de compartir carpetas entre equipos. Dependiendo del contexto y de los sistemas involucrados, se utilizan diferentes protocolos y herramientas. Los escenarios más comunes incluyen:

  • Entre equipos Linux: Usar herramientas como NFS (Network File System) o SSHFS.
  • Con equipos Windows: Utilizar Samba para permitir la interoperabilidad entre Linux y Windows.
  • En un servidor: Usar NFS o Samba para compartir archivos con múltiples usuarios o máquinas en una red.

Importancia de compartir archivos en la red

Compartir archivos en la red permite que múltiples usuarios puedan acceder a la misma información de manera simultánea. Esto es esencial en entornos de trabajo colaborativos donde varias personas necesitan acceder a documentos, configuraciones o archivos de trabajo de forma centralizada. Además, la centralización de archivos en un servidor reduce la redundancia de datos y facilita las copias de seguridad.

Diferentes escenarios de uso

Los escenarios más comunes de uso de la compartición de carpetas incluyen:

  • Entre sistemas Linux: Se puede usar NFS (Network File System) o SSHFS (SSH File System).
  • Entre Linux y Windows: Usando Samba, que permite la interoperabilidad entre ambos sistemas operativos.
  • Para servidores de archivos: Configurar NFS o Samba para centralizar los archivos accesibles por múltiples usuarios o máquinas en la red.

Ejemplo práctico

Configuración de Samba en Linux para compartir carpetas con Windows

A continuación, se muestra cómo configurar Samba para compartir una carpeta entre un sistema Linux y un equipo Windows:

Paso 1: Instalación de Samba

sudo apt update
sudo apt install samba

Paso 2: Crear una carpeta para compartir

mkdir /home/usuario/compartido

Establecer los permisos adecuados para la carpeta compartida:

chmod 777 /home/usuario/compartido

Paso 3: Configurar Samba

Editar el archivo de configuración de Samba:

sudo nano /etc/samba/smb.conf

Agregar la configuración de la carpeta compartida:

[compartido]
   path = /home/usuario/compartido
   browseable = yes
   read only = no
   guest ok = yes

Paso 4: Crear un usuario de Samba

Para crear un usuario de Samba, usa el siguiente comando:

sudo smbpasswd -a usuario

Paso 5: Reiniciar el servicio Samba

Reinicia el servicio de Samba para aplicar los cambios:

sudo systemctl restart smbd

Paso 6: Acceder desde Windows

Desde el equipo Windows, abre el explorador de archivos y escribe en la barra de direcciones:

\\IP_DEL_SERVIDOR\compartido

Ingresa las credenciales si es necesario para acceder a la carpeta compartida.

Introducción

¿Qué es un usuario?

un usuario es una entidad (persona o proceso) que interactúa con un sistema, generalmente a través de una cuenta con un identificador único. Los usuarios tienen permisos específicos que determinan qué acciones pueden realizar dentro del sistema, como acceder a archivos o ejecutar programas.

¿Diferencias entre usuarios normales y root?

En sistemas operativos como Linux, los usuarios normales tienen privilegios limitados, lo que significa que no pueden realizar tareas administrativas como instalar software o modificar configuraciones críticas del sistema. Por otro lado, el usuario 'root' tiene privilegios de superusuario, permitiéndole realizar cualquier acción en el sistema, lo que lo convierte en un usuario con acceso total a todos los recursos del sistema.

Importancia de la gestión de usuarios

La gestión de usuarios es fundamental para mantener la seguridad y el control de un sistema. A través de una adecuada gestión de cuentas, se pueden asignar permisos específicos a los usuarios, controlar el acceso a recursos y evitar que personas no autorizadas realicen acciones que puedan poner en peligro la integridad del sistema. Esto también es esencial para la administración de sistemas multicanal, donde se deben manejar distintos niveles de acceso para cada usuario o grupo.

Crear un usuario

Para añadir un nuevo usuario al sistema:

sudo useradd -m nombre_usuario

Para establecer la contraseña del usuario:

sudo passwd nombre_usuario

Modificar un usuario

Saber en que usuario estás conectado

whoami

Saber que usuarios estan conectados al equipo

who

Para conocer el ID del usuario al que estas conectado

id -u

Para cambiar el nombre de usuario:

sudo usermod -l nuevo_nombre antiguo_nombre

Para agregar un usuario a un grupo:

sudo usermod -aG grupo nombre_usuario

Para cambiar la contraseña de un usuario

sudo passwd nombre_usuario

Eliminar un usuario

Para eliminar un usuario sin eliminar su directorio personal:

sudo userdel nombre_usuario

Para eliminar un usuario y su directorio personal:

sudo userdel -r nombre_usuario

Listar usuarios

Para ver la lista de usuarios en el sistema:

cat /etc/passwd

Modificar Permisos de un Archivo

Para otorgar permisos específicos a un usuario sin cambiar la propiedad del archivo, usa ACLs:

sudo setfacl -m u:usuario:rwx archivo

Para ver los permisos ACL de un archivo:

getfacl archivo

Para revocar los permisos de un usuario específico:

sudo setfacl -x u:usuario archivo

Explicación:

-m u:usuario:rwx: Asigna permisos de lectura, escritura y ejecución a un usuario específico.getfacl archivo: Muestra los permisos ACL actuales del archivo. -x u:usuario: Elimina los permisos ACL de ese usuario en el archivo.
